✅ Answer Key (With Questions)

1. What are data structures?
✔ B. Ways to store and organize data efficiently

2. Why are data structures important?
✔ C. For interviews, CS subjects, and real-world applications

3. What is an array?
✔ C. An ordered collection stored in contiguous memory

4. What does zero-based indexing mean?
✔ B. Indexing starts from 0

5. What is a major advantage of arrays?
✔ C. Fast access of elements

6. Time complexity for accessing an element in an array is:
✔ C. O(1)

7. What is a disadvantage of arrays?
✔ B. Slow insertion and deletion

8. What happens when an element is inserted in the middle of an array?
✔ C. Elements shift and memory may be reallocated

9. What is a linked list?
✔ C. A collection where each element points to the next

10. How are linked list elements stored in memory?
✔ C. Non-contiguous locations connected by pointers

11. Main advantage of linked lists over arrays is:
✔ C. Faster insertion and deletion

12. What is a disadvantage of linked lists?
✔ C. Slow access/search

13. Which structure provides fast access, and which provides fast insertion/deletion?
✔ B. Array – access, Linked list – insert/delete

14. What is a hash map?
✔ C. Key-value pair data structure

15. Which of the following is true about hash maps?
✔ C. Fast searching using keys

16. Average time complexity of search in a hash map is:
✔ C. O(1)

17. What are hash maps called in Python?
✔ D. Dictionaries

18. Which data structure follows LIFO?
✔ B. Stack

19. Which operation removes the top element from a stack?
✔ C. Pop

20. Which real-life example represents a stack?
✔ C. Stack of plates

21. Which data structure follows FIFO?
✔ C. Queue

22. What does enqueue mean in a queue?
✔ B. Add element at back

23. What is a tree data structure?
✔ B. Hierarchical structure with nodes and edges

24. What is the topmost node of a tree called?
✔ C. Root

25. What is a Binary Search Tree property?
✔ B. Left < Parent < Right

26. Time complexity of searching in a balanced BST is:
✔ C. O(log n)

27. Why are trees efficient for searching?
✔ B. They eliminate half the data each step

28. What is a graph?
✔ C. Nodes and edges representing connections

29. Which feature is NOT mandatory in trees but allowed in graphs?
✔ C. Cycles

30. Which is a real-life application of graphs?
✔ B. Uber route optimization

31. Why are graphs considered difficult?
✔ C. Many variations and fewer restrictions

32. Correct order from easiest to hardest data structures is:
✔ B. Array → Linked List → Hash Map → Stack/Queue → Tree → Graph
